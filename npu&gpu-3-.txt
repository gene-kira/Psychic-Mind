🧩 What each subsystem does
1. ReplicaNPU (your custom NPU)
This is a software-simulated neural processor with:
• 	MAC units
• 	Vector MAC
• 	Matrix multiplication
• 	Activation units
• 	Plasticity
• 	Integrity checks
• 	Predictive heads
• 	Symbolic bias
• 	Instruction scheduler
• 	Temporal memory
• 	Energy + cycle accounting
• 	Serialization
This organ behaves like a hardware NPU, not a neural network.
It simulates how a real NPU would process vectors, tensors, and predictions.
You now have two NPU layers:
• 	ReplicaNPU → hardware‑style, symbolic, predictive
• 	Torch NPUOrgan → neural network controller
• 	NPURaidEnsemble → multiple NPUOrgan fused
This gives you a hybrid NPU architecture.

2. GPU Organ
This organ reads:
• 	GPU utilization
• 	Memory utilization
• 	VRAM fraction
• 	Temperature
• 	Power usage
• 	Power limit
It also attempts to set power limits (may fail on consumer GPUs).
This gives the NPU real‑time hardware telemetry.

3. Scheduler
This is the autonomic nervous system.
It:
• 	Reads GPU telemetry
• 	Feeds it into the NPU RAID
• 	Gets performance + power scaling predictions
• 	Chooses a mode (COAST, NORMAL, AGGRESSIVE, BEAST)
• 	Applies GPU policy
• 	Broadcasts state to:
• 	Local subscribers
• 	Swarm mesh
This is the brainstem of the organism.

4. Policy Bus (Socket Server)
A TCP server that:
• 	Streams JSON policy packets to any connected client
• 	Allows other processes to subscribe
• 	No polling needed
This is your local IPC bus.

5. Swarm Sync Organ
A UDP multicast mesh that:
• 	Broadcasts your node’s state
• 	Receives other nodes’ states
• 	Tracks peers
• 	Builds a distributed cluster
This turns your system into a multi‑node AI swarm.

6. GUI Cockpit
A Tkinter dashboard that shows:
• 	GPU stats
• 	NPU RAID stats
• 	Mode
• 	Power scaling
• 	Swarm peers
• 	Node ID
• 	NPU discovery results
This is your operator cockpit.
