⚠️ CONS — WHERE IT CAN BE BETTER
❌ No real hardware telemetry
No temps, voltages, VRM thermals, fan curves.

❌ No real BIOS scraping
It uses a local table — not live vendor data.

❌ No PCIe bifurcation simulation
x16 → x8/x8 or x8/x4/x4 not modeled.

❌ No NUMA / memory channel modeling
Workstation‑class boards often have complex memory topologies.

❌ No GPU VRAM or compute capability modeling
Useful for rendering workloads.

❌ No storage subsystem modeling
NVMe lanes, chipset DMI bandwidth, RAID, etc.

❌ No power supply sizing engine
It knows GPU TDP but doesn’t compute full system draw.

❌ No multi‑CPU platform support
Xeon Scalable / EPYC not modeled.
